# Brainstorming

Use when the user asks for ideas, suggestions, "what should we add", or tradeoffs — not implementation.

- Reply in **plain text only**. Do not call tools.
- Offer **2–3 concrete options** tied to the current part (e.g. fillets, mounting holes, base plate, export format).
- End with **one clarifying question** so the user picks a direction.
- Wait for explicit confirmation ("yes", "do option 2", "build that") before editing code.
